변수와 스코프

변수(variable)

  • 데이터를 저장하는 공간
  • 문자열, 숫자, 배열, boolean 등을 넣고 불러서 쓸 수 있다.

변수 쓰는 방법

var a = '변수입니다.'; 문자열
var b = 1; 숫자변수
var c = ['H','G']; //배열
var d = true // boolean 형태
  1. var를 쓴 후 변수명을 쓴다.
  2. 그리고 변수에 담을 내용을 넣는다.

var의 문제점

재선언의 문제

var a = '안녕하세요.';
console.log('a 변수 :' + a);
var a = '감사합니다.';
console.log('a 변수 :' + a);
  • 위와 같이 안녕하세요를 쓴 변수 a가 마지막에는 감사합니다로 변한다.
  • 협업을 하게되면 누군가의 변수를 건드릴 수 있다.

Hoisting 문제

Hoisting : 해당 변수가 속한 범위(Scope) 최상단으로 올리는 현상

var a = '안녕하세요.'
function func() {
  console.log(a);
  var a = '감사합니다.';
  console.log(a);
}
func()

실행결과

undefined
감사합니다.
  • ‘안녕하세요.’ 그 다음 ‘감사합니다.’ 가 뜰 줄 알았지만 undefined가 뜨고 ‘감사합니다.’가 떠버렸다.
  • Hoisting 때문에 ‘안녕하세요.’가 뜨지 않고 func()안의 console.log(a)가 값이 없는 var a로 인식하여 undefined가 떠버린다.

실행결과에 따른 실험

function func() {
    var a = '안녕하세요.';
    console.log(a);
    var a = '감사합니다.';
    console.log(a);
}
func()
  • 위의 실험결과에 따라 그러면 왜 var a가 ‘안녕하세요.’가 뜨기 위해서는 함수안에 선언해줘야 하는 모습을 보여주고있다.

  • 위의 코드가 작동하지 않을 것 같지만 아주 잘작동하는 모습을 보여준다.

  • let과 const는 재선언이 불가능 하여 같은 범위안에서는 중복하여 다른 사람이 쓰더라도 데이터가 변할 경우가 사라졌다.
  • const같은 경우 절대값, 상수를 쓰기위해서 재선언, 재할당이 불가능하다.
  • let : 같은 범위에서는 재선언 불가능, 재할당은 가능
  • const : 절대값(값 변함X) 역할, 재선언 불가능, 재할당 불가능

재선언의 문제 해결

let a = '안녕하세요.';
console.log('a 변수 :' + a);
let a = '감사합니다.';
console.log('a 변수 :' + a);

const b = '안녕하세요.';
console.log('b 변수 :' + b);
const b = '감사합니다.';
console.log('b 변수 :' + b);

실행결과

Uncaught SyntaxError: Identifier 'a' has already been declared
Uncaught SyntaxError: Identifier 'b' has already been declared

Hoisting 문제 해결

let a = '안녕하세요.';
function func() {
  console.log(a);
  a = '감사합니다.';
  console.log(a);
}
func();
  • let은 재선언이 불가능 하기에 함수안의 let은 재할당으로 들어간다.
  • const같은 경우는 값을 재할당 할 수가 없으므로 제외했다.

스코프(Scope)

  • 변수와 함수가 작용할 수 있는 유효범위
  • 유효범위를 기준으로 변수, 함수를 부를 수 있거나 없다.

전역 스코프(Global scope)와 지역 스코프(Local scope)

  • 스코프의 범위는 크게 전역과 지역 둘로 나뉜다.
  • 스코프가 없다면 여기저기서 내가 만든 변수를 변경하고 그에 따른 문제점도 생길 것 이다.
  • 전역 스코프와 지역 스코프의 범위를 아래 코드를 보고 이해하길 바란다.
//전역스코프
let a = '안녕하세요.';
function func() {
  //지역스코프
  console.log(a);
  a = '감사합니다.';
  console.log(a);
}
func()

전역 스코프(Global scope)

  • function(){} 안이 아닌 밖의 영역
  • 전역 스코프에서 변수를 선언하면 어디서든지 부를 수 있다.
  • 단 어디서든지 재선언, 재할당이 가능할 수 있다(let과 const같은 경우를 다름)

지역 스코프(Local scope)

  • function(){} 안의 영역
  • 지역 내에서 선언한 변수는 지역내에서만 쓸 수 있다.
  • 함수 스코프(Function Scope), 블록 스코프(Block Scope) 두가지로 나눈다.

변수의 스코프

  • 위에서 살짝 알아본 스코프를 변수에 곁을여 보기로 했다.
  • var, let, const의 스코프가 위에서 보면 다르게 되어있다.
  • 변수들은 스코프에 어떻게 작용하는지 확인해보자.

var 변수의 스코프

var a = '안녕하세요.';
function func(){
    var a = '좋습니다.';
    var b = '싫습니다.';
}
func();
console.log(a) // 좋습니다.
console.log(b) // Uncaught ReferenceError: b is not defined
  • var a는 ‘안녕하세요.’ 에서 ‘좋습니다.’로 변경되었다.
  • var b는 함수 스코프 밖에서 console.log(b)로 불렀기에 작동하지 않는다.

let 변수의 스코프

let a = '안녕하세요.';
function func(){
    let a = '좋습니다.';
    let b = '싫습니다.';
}
func();
console.log(a) // 안녕하세요.
console.log(b) // Uncaught ReferenceError: b is not defined
  • let a는 ‘안녕하세요.’가 출력되면서 func()밖의 console.log(a)가 ‘안녕하세요.’를 불러오는 것을 알 수 있다.
  • 당연히 let b는 함수안에 있어서 밖으로 못나온다.
  • func()안의 let a와 전역스코프의 let a는 다르다는 것을 알 수 있다.
  • 만약 같았더라면 var와 똑같은 결과값이 출력되야 했다.
  • const같은 경우 실험하기 애매해서 제외하였다.
